# 1. 项目概述

海外文物知识服务子系统是平台面向普通用户、研究者和课程验收人员的 Web 端服务入口，主要基于知识图谱构建子系统提供的文物数据，为用户提供文物浏览、检索查询、图谱与统计可视化、用户登录注册和个人中心等功能。系统界面设计参考国际一流博物馆网站的浏览与检索体验，强调数据可读性、交互清晰度和服务稳定性。

本项目管理计划用于明确 5 周开发周期内的范围、分工、进度、质量要求、风险应对和交付物，确保各成员围绕统一前端框架与接口规范协同开发，并能在课程验收前完成可运行、可演示、可部署的知识服务子系统。

# 2. 项目目标与成功标准

- 完成 Web 端基础框架、首页、路由、用户登录注册和用户中心，为各功能模块提供统一入口。

- 实现数据浏览功能，支持列表/卡片视图切换、多维筛选、排序、文物详情、相关文物推荐和文物对比。

- 实现数据查询功能，支持关键字检索、高级组合查询、知识图谱关系查询、分页、二次筛选和查询结果导出。

- 实现数据可视化功能，包含知识图谱关系图、文物时间轴、地理分布图和统计分析看板。

- 完成服务器部署，保证前端页面、接口配置、静态资源、跨域配置和演示环境稳定可用。

- 系统在主流桌面浏览器中可正常访问，核心页面无明显布局错乱，主要接口异常时有明确提示。

# 3. 项目范围

## 3.1 必做范围

- 框架与公共模块：Vue 3 + Vite 项目结构、Element Plus 全局配置、Axios 请求封装、Pinia 状态管理、Vue Router 路由守卫和公共布局。

- 首页与用户模块：首页导航、模块入口、登录、注册、退出登录、用户信息展示与编辑。

- 数据浏览：文物列表、文物卡片、筛选排序、分页、详情页、图片查看、关联实体展示、推荐列表和 2-3 件文物对比。

- 数据查询：简单查询、高级查询、知识图谱查询、结果二次筛选、结果分页和 CSV/JSON 导出。

- 数据可视化：基于 ECharts / AntV G6 展示图谱关系、时间轴、地图分布和统计看板。

- 部署与联调：生产构建、服务器部署、反向代理或静态资源托管、接口地址配置、环境变量管理和上线检查。

## 3.2 可选增强范围

- 浏览记录、收藏夹分组、评论点赞、个性化推荐等用户个人服务功能。

- 查询条件保存、热门搜索词、低匹配结果提示和高级查询模板。

- 图谱节点按需展开、关系路径高亮、地图与列表联动、可视化截图导出。

## 3.3 不在本阶段范围

- 不负责海外博物馆数据爬取、清洗、实体对齐和知识图谱底层构建，但需要对接其数据结果。

- 不负责知识问答子系统的大语言模型问答能力，仅保留跳转入口或接口预留。

- 不负责移动端 App 的 HarmonyOS 页面开发。

# 4. 技术路线与系统架构

前端采用 Vue 3 + Vite 作为基础框架，Element Plus 提供表格、表单、分页、弹窗、上传、菜单和消息提示等通用组件；Axios 统一管理 API 请求、响应拦截和错误处理；Pinia 保存用户信息、筛选条件、查询状态、可视化配置和收藏数据；Vue Router 负责首页、浏览、查询、可视化、详情页、用户中心和登录注册页面之间的跳转；ECharts / AntV G6 分别承担统计图表、地图、时间轴和知识图谱关系图展示。

| **层级** | **主要内容** | **关键输出** |
|----|----|----|
| 页面展示层 | 首页、浏览页、查询页、可视化页、详情页、用户中心、登录注册页。 | Vue 页面、组件、路由配置 |
| 状态与服务层 | 用户状态、筛选条件、查询参数、图表配置、接口请求和异常处理。 | Pinia store、Axios API 模块 |
| 数据接口层 | 对接文物列表、详情、筛选字典、查询、图谱关系、统计指标和用户接口。 | 接口文档、Mock 数据、联调记录 |
| 部署运行层 | Vite 构建产物、服务器配置、静态资源托管、接口代理和环境变量。 | 部署说明、运行环境、访问地址 |

# 8. 质量管理计划

- 需求质量：所有页面和功能点必须能对应课程说明中的浏览、查询、可视化和用户个人信息管理要求。

- 代码质量：组件命名、路由命名、API 模块和 Pinia store 保持统一规范，避免重复实现公共请求和状态逻辑。

- 交互质量：列表、筛选、查询、图表、登录注册等核心流程必须包含加载、成功、空结果和错误状态。

- 数据质量：文物名称、年代、类型、材质、博物馆、图片、详情 URL 和图谱关系字段展示前需要做空值保护。

- 性能质量：首页和列表页避免一次性加载过多数据；图谱关系图需要限制默认节点数量，支持按需展开。

- 兼容性质量：至少在 Chrome 或 Edge 中完成主要流程测试，保证常见桌面分辨率下布局稳定。

# 10. 风险管理

| **风险** | **影响** | **应对措施** | **负责人** |
|----|----|----|:--:|
| 后端或图谱接口延迟交付 | 前端模块无法按真实数据联调。 | 第 1 周确定 Mock 数据格式；第 3 周开始逐步替换真实接口。 | 李欣雨、吴蕊 |
| 文物字段缺失或格式不统一 | 筛选、详情和图表展示不稳定。 | 前端统一空值处理，接口文档明确字段类型和默认值。 | 张若焓、申宜卡 |
| 图谱可视化节点过多 | 页面卡顿或图谱不可读。 | 默认展示核心节点，支持搜索、筛选和点击展开。 | 李嘉辰 |
| 部署环境配置不一致 | 本地可运行但服务器不可访问。 | 记录 Node 版本、构建命令、接口地址和代理配置，部署后进行回归测试。 | 吴蕊 |
| 成员模块风格不一致 | 系统整体体验割裂。 | 统一公共布局、组件封装、颜色规范和页面间距，合并前进行集成检查。 | 李欣雨 |

# 11. 沟通与变更管理

- 每周至少进行一次进度同步，说明已完成内容、当前问题、下周计划和需要他人配合的接口或页面。

- 涉及接口字段、路由路径、公共组件和全局样式的变更，需要在小组内同步后再合并。

- 需求调整按“必做功能优先、选做功能后置”的原则处理，优先保证浏览、查询、可视化、登录注册和部署验收。

- 缺陷统一记录问题描述、复现步骤、负责人、优先级、状态和关闭时间，避免口头遗漏。
